# 红杉软件工作室

红杉是世界上长的最高的植物之一，已知最高的红杉有115米高，其根系可以扎入地下150米左右的地方。同时红杉也是地球上最长寿的植物之一，目前已知最老的红杉大约有2200岁。

我有幸听到一位老师的课，他讲到如何成为某个领域的专家时说道：选择一个领域，专注的去做，在这个领域做到一米宽，一百公里深的时候，你就成为这个领域的专家了。

这番话让我第一个想到的就是红杉这种植物，努力在一个领域里面深耕，服务好每一个客户，而不是只图赚快钱。与之相比，信誉和口碑才是最重要的。

# 我们是谁

程序员圈里有一个自嘲叫“35岁魔咒”，意思是程序员到35岁左右会很难找到工作。实际情况可能不尽相同，但是这个现象是客观存在的。正当壮年的30岁出头，很多人都已经有了家庭有了孩子，可能有相当一部分人还有债务压力，却在赖以生存的职业生涯上被年龄给“诅咒”了，当真是地狱笑话。

我们的团队主要就是由这样一群人组成的。平均年龄30岁上下，经历过职场的甩锅背锅、996的锤炼。遇见过只为KPI工作的同事、不懂实际业务但是一味迎合管理层只会逼大家加班的领导。最后看淡了职场的尔虞我诈，只想沉浸在技术中安安静静的写代码。直到有一天幡然醒悟，为什么我们不“只工作，不上班”呢？很快我的这个想法得到了技术圈几个朋友的认同，我们组了一个小团队在工作之余开始接触一些有技术开发需求的客户，这个小团队就是工作室的前身。

到目前为止，我们的团队已经具备了完整的项目开发能力，拥有有项目经理、UI设计、前端开发、Java开发、PHP开发和大数据架构师，可以胜任大部分业务产品的开发。

让我决定不再上班，全职做这个事情的契机是2024年3月份的公司裁员。这个时候我还是公司DevOps产品研发部门的前端经理，我的团队最多的时候有18个人。我不想讲我在职的3年间做出了哪些事情，因为这些无论从什么角度讲都有吹嘘的嫌疑。我只讲一点，当部门同事知道我要走的时候，都要请我吃饭，一周多的时间吃了十几顿，最后因为时间实在排不开婉拒了三个同事的邀请。申明一下我不是说自己的人缘有多好，只是想从侧面佐证一下我给这个部门和团队提供的价值肯定是得到认可的。

因为公司战略的调整，研发资源要向交付倾斜换言之以后大家都可能要去客户现场驻场开发。同时直接领导私下找我谈了几次，明里暗里说我的薪资太高，团队要降成本所以可能存在降薪的情况，让我有心里准备。

思量再三，觉得是时候离开了，就跟领导申请了一个裁员名额，拿了大礼包走人。因为家庭、孩子的综合原因，决定不再找工作上班了，开始做点自己的事情。

# 零工经济会成为企业用工的新形势

做出“不上班，只工作”这个决定的原因，是我认识到在当前互联网技术发展的背景下，零工经济有很大发展空间。这让一些有研发需求但是不愿养研发团队公司能够有效的节约成本。传统的研发人力外包公司和建站公司的收费往往又让一些小企业望而却步，这些因素综合起来的结论就是，市场上存在一些需求还没有被满足。零工经济在国内又是刚起步，我认为零工经济一定会在不远的将来成为中小型企业用工的新形势，我们这一群30岁以上的“老程序员”可以满足这个需求。

# 我们如何满足客户需求

首先，我们的团队成员有着丰富的项目开发经验，除了几个核心成员外，其他新进成员都是通过推荐加入的，推荐的门槛就一个：靠谱。推荐后会让对方给简历，我们会做一个简单的面试来确认对方的技术能力和沟通能力以及人品性格是否符合我们团队的风格。

其次，我们团队成员分为签约和兼职，签约的成员都是全职参与项目开发，如果资源不够就会召唤兼职的小伙伴加入。我们有严格的项目管理制度，把功能点分割成一定到颗粒度，保证开发者可以在我们要求的时间做到我们要求的样子。也就是这样的项目管理，可以让所有参与开发的人员随时可以进来，随时可以出去。而不会出现某个人退出整个项目延期的情况。

在项目开发中，我们遵守标准的敏捷开发流程，每个迭代开发测试环节一个不少，使用在线项目管理工具，客户随时可以看到项目进度。

# 一些愿景

希望经过我们的努力，能够在一年之后成功发展10个左右能够长期合作的客户，以服务质量获取行业口碑。
